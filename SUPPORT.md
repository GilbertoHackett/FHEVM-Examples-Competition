## ‚úÖ Before Asking for Help

1. **Read the Documentation**
   - Check [GETTING_HELP.md](./GETTING_HELP.md) for navigation
   - Find relevant guide
   - Read thoroughly

2. **Search Existing Resources**
   - Search [FAQ.md](./FAQ.md)
   - Check [TROUBLESHOOTING.md](./TROUBLESHOOTING.md)
   - Look at [EXAMPLES_CATALOG.md](./EXAMPLES_CATALOG.md)
   - Search GitHub issues

3. **Try to Debug**
   - Add console.log statements
   - Run minimal example
   - Check error messages carefully
   - Try steps from troubleshooting guide

4. **Prepare Good Information**
   - Your environment details (OS, Node.js version)
   - Exact error message
   - Steps to reproduce
   - What you've already tried

## üìù How to Report Issues

### Bug Reports
**File**: Use [bug_report template](.github/ISSUE_TEMPLATE/bug_report.md)

**Include**:
- Clear description of bug
- Steps to reproduce
- Expected vs actual behavior
- Environment (OS, versions)
- Error message/stack trace
- What you've tried

**Link**: [Report bug on GitHub](https://github.com/yourusername/fhevm-examples/issues/new?template=bug_report.md)

### Security Issues
**Method**: DO NOT create public issue

**Instead**:
1. Read [SECURITY_POLICY.md](./SECURITY_POLICY.md)
2. Email: [security@example.com](mailto:security@example.com)
3. Include description, impact, suggested fix

**Response**: 24-48 hours

### Feature Requests
**File**: Use [feature_request template](.github/ISSUE_TEMPLATE/feature_request.md)

**Include**:
- Problem description
- Proposed solution
- Use case
- Alternatives considered
- Category (example/documentation/tool)

**Link**: [Request feature on GitHub](https://github.com/yourusername/fhevm-examples/issues/new?template=feature_request.md)

## üí¨ Communication Guidelines

### When Asking Questions

‚úÖ **Do**:
- Be specific and detailed
- Include error messages (full text, not just a line)
- Share relevant code (in code blocks)
- Explain what you've tried
- Be respectful and patient

‚ùå **Don't**:
- Post vague error descriptions
- Share sensitive information (private keys, passwords)
- Demand immediate response
- Be disrespectful or angry
- Post the same question multiple times

### Example Good Question
```
Title: "Counter example fails to compile with euint32"

I'm trying to run the Counter example but it fails to compile.

Environment:
- OS: macOS 13.1
- Node.js: 20.10.0
- npm: 10.2.3

Steps:
1. npm install
2. npm run compile

Error:
```
TypeError: euint32 is not defined at line 15 of Counter.sol
```

What I've tried:
- Checked import statements (they look correct)
- Cleared node_modules and reinstalled
- Checked that @fhevm/solidity is installed

Thank you for any help!
```
